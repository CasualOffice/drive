## Bind & origins

| Var | Default | Notes |
|---|---|---|
| `DOCHUB_BIND` | `127.0.0.1:8080` | TCP bind. Use `0.0.0.0:8080` behind a proxy. |
| `DOCHUB_APP_ORIGIN` | — | Required. `https://hub.example.com`. Cookies + CSP. |
| `DOCHUB_USERCONTENT_ORIGIN` | — | Required. `https://usercontent-dochub.example.com`. Must differ from app. |
| `DOCHUB_TRUST_X_FORWARDED` | `false` | Only enable behind a trusted proxy you control. |
| `DOCHUB_LOG_LEVEL` | `info` | `error`, `warn`, `info`, `debug`, `trace`. |

## Storage backend

`DOCHUB_STORAGE_BACKEND` picks one of: `fs`, `s3`, `minio`, `memory`. Whatever you choose, document bytes are AES-256-GCM encrypted by `dochub-crypto` **before** they reach the backend — no plaintext is ever written to storage.

### Filesystem (`fs`)

| Var | Default | Notes |
|---|---|---|
| `DOCHUB_FS_ROOT` | — | Absolute path. Storage keys are ULIDs; the adapter root-confines every resolved path. |

### S3 (`s3`)

| Var | Default | Notes |
|---|---|---|
| `DOCHUB_S3_BUCKET` | — | Required. |
| `DOCHUB_S3_REGION` | `us-east-1` | |
| `DOCHUB_S3_ACCESS_KEY_ID` | — | IAM-scoped to the bucket. |
| `DOCHUB_S3_SECRET_ACCESS_KEY` | — | |
| `DOCHUB_S3_ENDPOINT` | (AWS) | Set for non-AWS providers (Backblaze B2, Cloudflare R2). |

BYO-bucket credentials are themselves sealed with the same envelope scheme before they are persisted — see [Architecture](/docs/architecture).

### MinIO (`minio`)

Same as `s3` plus an explicit endpoint and forced path-style addressing:

```
DOCHUB_STORAGE_BACKEND=minio
DOCHUB_S3_BUCKET=dochub-prod
DOCHUB_S3_REGION=us-east-1
DOCHUB_S3_ENDPOINT=https://minio.internal:9000
DOCHUB_S3_ACCESS_KEY_ID=…
DOCHUB_S3_SECRET_ACCESS_KEY=…
```

### Memory (`memory`)

For testing only. Loses everything on restart. No env vars.

## Encryption & keys

Encryption is not optional. The boot sequence **refuses to start** without a master KEK or a configured KMS.

| Var | Default | Notes |
|---|---|---|
| `DOCHUB_MASTER_KEY` | — | Required unless `DOCHUB_KMS_*` is set. 32-byte base64 KEK that wraps every per-workspace data key (DEK). Never logged. Rotating it re-wraps DEKs without rewriting document blobs. |
| `DOCHUB_KMS_PROVIDER` | — | Optional. `aws`, `gcp`, or `hub` (HashiCorp). Delegates KEK custody to an external KMS instead of `DOCHUB_MASTER_KEY`. |
| `DOCHUB_KMS_KEY_ID` | — | Required when `DOCHUB_KMS_PROVIDER` is set. The external key handle. |

Per-workspace DEKs are stored wrapped, never plaintext. Losing the KEK (or KMS access) means the workspace data is unrecoverable — that's the point.

## Database

| Var | Default | Notes |
|---|---|---|
| `DOCHUB_DATABASE_URL` | `sqlite::memory:` | Production: `sqlite:///data/hub.db` or `postgres://…`. |

The schema is portable across SQLite + Postgres — every migration ships in `crates/dochub-db/migrations/` (TEXT ULIDs, ISO-8601 UTC, INTEGER bools; no JSONB/enum/native-UUID).

## Sessions & secrets

| Var | Default | Notes |
|---|---|---|
| `DOCHUB_SESSION_SECRET` | (random) | 32+ random bytes. Set this so sessions survive restarts. |
| `DOCHUB_SESSION_TTL_HOURS` | `168` | Browser cookie lifetime. |
| `DOCHUB_SESSION_COOKIE_NAME` | `__Host-dh_sid` | Don't change unless you have a reason. |
| `DOCHUB_HMAC_SECRET` | (random) | 32+ random bytes. Signs editor access tokens and fs/mem signed-URL `/raw/{token}` grants. Set it so tokens survive restarts. |

## OIDC (optional)

Argon2id passwords work out of the box; OIDC layers single sign-on on top via Authorization Code + PKCE against any compliant IdP.

| Var | Default | Notes |
|---|---|---|
| `DOCHUB_OIDC_ISSUER` | — | Discovery URL of the IdP. Enables OIDC when set. |
| `DOCHUB_OIDC_CLIENT_ID` | — | Registered client id. |
| `DOCHUB_OIDC_CLIENT_SECRET` | — | Registered client secret. |
| `DOCHUB_OIDC_REDIRECT_URL` | `${DOCHUB_APP_ORIGIN}/auth/oidc/callback` | Must match the IdP registration. |

## Rate limits

| Var | Default | Notes |
|---|---|---|
| `DOCHUB_UPLOAD_RPS` | `30` | Per-user upload requests per minute. |
| `DOCHUB_AUTH_LOCKOUT_THRESHOLD` | `5` | Failed sign-ins before short backoff. |

## Backup

Three things to back up, and they must be restored together to a consistent moment:

1. **The database.** SQLite: snapshot the `.db` file (use `sqlite3 .backup` for a hot snapshot). Postgres: `pg_dump`. Holds metadata, the hash chains, the audit log, and the wrapped DEKs.
2. **The storage backend.** `fs`: `rsync` the `DOCHUB_FS_ROOT`. `s3`/`minio`: enable bucket versioning + cross-region replication. Holds the encrypted document blobs.
3. **The master key / KMS access.** Without the KEK, backups 1 and 2 are ciphertext. Store it separately, under stricter custody than either.

The hub doesn't lock storage during a DB dump (that would freeze uploads); use the append-only audit log to pick a quiet window.

## Health & metrics

- `GET /healthz` — 200 always. Cheap.
- `GET /readyz` — 200 if storage + DB respond and the KEK is loaded.
- Logs are structured JSON when `DOCHUB_LOG_FORMAT=json`; otherwise pretty-printed. Keys and document bytes never appear in logs.
- Prometheus metrics endpoint is **not** in v0; planned for a later phase.
